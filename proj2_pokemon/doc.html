Reflection:

As fun and intriguing this project was, it was also pretty taxing on me. I believe the biggest obstacle that I struggled with most on this project was not devoting as much time as I wanted to on this project. While I am proud of the end product I was able to create, I feel like I coulda added a little bit more to it, just to give it a little more 'spice' to it. The biggest reason why this project was a bit stressful was largely due to other projects I was working on at the time. I feel like my attention was split thinly between various academic committments, so much so where I was unable to truly focus on this particular project.

That being said, what I did produce, I belive to be a very nice representation of simple API manipulation and diplay. This simple Pokemon Finder works simply by using a Pokemon's name or pokedex number and displays some brief info on it. Using try/catch statements, I was even able to prevent my web app from crashing and instead display a default screen with no pokemon being displayed. Additionally, all three unique filters: shiny, gender, and mega filters all seem to work amazingly!

Tip when using the Gender filter: not all pokemon have visual differences between gender. Try searching "Pikachu", "Venusaur", and/or "Rhydon" to see this filter in play. Same goes for the mega filter; not all pokemon have mega forms. Using terms like "Venusaur", "Blaziken", and/or "Aggron" are sure to work!

In terms of grade, at the very least, I believe I at least earned a B or B+ grade on my work. Despite my recent challenges and tribulations, I'm still very proud of how this project turned out and I believe with its functioning and simple and easy-to-use design, I believe my work is worthy of a B/B+ grade at the very least.
